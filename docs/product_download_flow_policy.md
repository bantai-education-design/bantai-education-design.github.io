# 商品ページ 販売導線 方針書

- 制定日: 2026-07-14
- 対象: Ban.Tai Education Design 公式HP（bantai-education-design.github.io）の全商品ページ、商品一覧ページ、トップページのCTA
- 関連文書: [現状調査報告書](product_download_flow_audit.md)

## 目的

　試用版の入手・購入・外部掲載の3つの導線について役割を明確に分け、利用者が迷わずダウンロード・購入できる商品ページに統一する。本方針は今後の商品ページ新設・改修すべてに適用する。

## 方針

1. 試用版は公式HPの商品ページから直接ダウンロードできる導線にする。試用版の入手のために外部サイト（BOOTH・Vector等）へ遷移させない。

2. 購入・ライセンス取得はBOOTHへ集約する。価格表示・決済・購入後のライセンス案内はBOOTHの商品ページを正とする。

3. Vectorは認知拡大・外部掲載用の補助導線として残す。Vector掲載ページへのリンクは掲載実績の紹介として扱い、購入・最新版入手の主導線にはしない。

4. 商品ページのボタン順は「試用版DL → BOOTH購入 → Vector掲載」の順に統一する。商品詳細ページ・商品一覧ページ・トップページのCTAすべてに同じ順序を適用する。

5. 試用版ZIPの置き場所は、基本は公式HP配下 /downloads/ とする。

6. 大容量・履歴管理が必要なファイルのみ GitHub Releases を併用できる。

7. GitHub Releases を使う場合も、利用者向けには公式HPの商品ページからダウンロードする導線として見せる。ページ上の見せ方は「公式HPからのダウンロード」で統一し、GitHubのリポジトリページやReleases一覧ページそのものへは誘導しない。

8. Vector専売・BOOTH未掲載の有料導線は原則作らない。有料製品を公開する際は、必ずBOOTHの購入導線を先に整えてから公開する。

9. 画像文字入れくんは当面、無料ツールまたは旧配布ツールとして整理し、BOOTH販売は別判断とする。それまでの間、方針8の例外として現状のVector導線を維持する。

10. 無料ツールと有料・試用版アプリをページ上で混同しない。無料ツールは「無料」バッジと「無料でダウンロードする」ボタン、有料製品は「10日間無料体験」等のバッジと「試用版DL → BOOTH購入」の導線で明確に区別する。

## 適用上の注意

- 本方針と矛盾する既存の記述（「ダウンロード導線はBOOTHに集約」「公式HPからZIPやEXEへの直接リンクは掲載していません」等）は、導線改修の際にあわせて削除・書き換える。対象箇所は現状調査報告書の第5節に一覧化している。
- 商品一覧ページ（products/index.html）は tools/build_products.py による生成物のため、ボタン順の変更は data/products.json とスクリプト側の改修で行い、生成物を直接編集しない。
- 本方針の制定時点では文書化のみを行い、HTML・data/products.json・build_products.py の修正、ZIP配置、Release作成は別作業として実施する。
