## 1. 証明写真メーカー（products/id-photo/index.html）

- 旧方針文言: 161行目「正式版はBOOTHの商品ページから購入・ダウンロードできます。公式ホームページから直接ZIPやEXEは配布しません。」後半の一文が方針1と不整合な明文。この製品は試用版を提供していないため当面の実害はないが、「正式版はBOOTHで購入できます」だけに書き換えるのが安全。
- id名の不一致: 158行目 section id="vector" の実体はBOOTH購入セクションで、Vectorへの掲載実績もリンクもない。navの「購入・ダウンロード」（38行目）がこの #vector を指している。id="purchase" 等への変更と、ページ内リンクの同時修正が必要（外部からのアンカーリンク流入は考えにくいが、変更時はnavとセットで行う）。
- 重複セクション: 「プレビュー機能」カード（81〜93行目と118〜130行目）が、バッジ・見出し・本文・画像までほぼ同一で2回出現。1つに統合すべき。
- 導線の現状: BOOTH購入（方針一致）→ note記事。Vectorリンクなし。試用版DL予定は現時点でないため、本文とボタンの食い違いはない。
- GA4タグ: あり。
- 修正方針案: 上記3点（文言書き換え・id整理・重複統合）は試用版配置と無関係に単独で修正できる。

## 2. はじめての五線紙メーカー（products/first-staff-paper/index.html）

- Gumroad表記: 176行目「最新プログラムのダウンロードや正規ライセンスキー申請手続きは、必ずこちらの公式HPおよびBOOTH/Gumroad経由で行ってください。」Gumroadは現行チャネルにないため削除する。出現はこの1箇所のみ。
- 本文とボタンの食い違い（このページ最大の問題）: 導入フローStep1（112行目）は「公式HP（本ページ）から無料体験版をダウンロードして」と新方針どおりの説明だが、実際のダウンロードボタン（186行目）は「10日試用版をBOOTHでダウンロード（0円）」でBOOTH行き。本文が先に新方針になっており、ボタンが旧方針のまま。完全な解消には試用版ZIP（88.3MB・GitHub Releases推奨）の配置が必要なため、配置フェーズとセットで修正する。
- Vector関連: 176行目「Vectorでは体験版の配布のみを行っています」という記述はあるがVectorへのリンクはない。方針3（補助導線）とは整合。Vector掲載ページのURLが確定していれば「Vector掲載ページを見る」ボタンの追加候補。
- ボタン順: 試用版（BOOTH 0円）→ 製品版購入（BOOTH 3,700円）で、順序自体は方針4に合致。DL先だけが問題。
- 重複セクション・id名の不一致: なし（id="download" は内容と一致）。
- GA4タグ: なし。
- 修正方針案: Gumroad削除は単独で即修正可能。ボタンの公式HP直DL化は試用版配置（第2段階B相当・Releases方式）とセット。

## 3. 教育計画システム（products/education-planning/index.html）

- 同一URLの3ボタン（このページ最大の問題）: 200〜204行目の dl-grid に「BOOTHからダウンロードする」ボタンが3つ並び、説明だけ「教務支援＋週案連携（約130MB）」「教育計画・時数・提出書類中心（約60MB）」「週案作成・印刷・集計中心（約70MB）」と異なるが、リンク先は3つとも同一のBOOTH商品（items/8547376）。さらに棚卸し（trial_zip_inventory.md）で判明したとおり、記載サイズは実在ZIP（フル版277〜833MB、SetupOnlyでも82MB）と大きく乖離している。
- 整理の方向: ボタンを1つに統合して「BOOTHからダウンロードする（バリエーションは商品ページで選択）」とするのが暫定案。構成別の正確な案内は、モニター版配布構成の再整理（5.70.7/5.70.8の版確定・構成確定）フェーズで行う（本調査の範囲外・別フェーズと確定済み）。
- 旧方針文言: 「直接配布しません」系の明文はなし。
- 導線の現状: 無料モニター登録（Googleフォーム）→ BOOTH DL → ライセンスキー自動送付、というモニター方式。navの「無料ダウンロード」の実体がBOOTH誘導である点は、モニター版の配布構成確定後に整合させる。
- 重複セクション・id名の不一致: なし。
- GA4タグ: あり。
- 修正方針案: 同一URL3ボタンの統合（暫定）は単独で修正可能。サイズ表記の修正は配布構成確定とセット。

## 4. 漢字練習帳（products/kanji-practice/index.html）

- 旧方針文言: なし。
- ボタン順の逆転: ヒーロー（47〜48行目）が「BOOTHで購入・ダウンロードする」（primary）→「10日間無料体験版」（#trialアンカー）の順で、方針4（試用が先）と逆。#trialセクション内のボタン（107行目）と最下部CTA（121行目）もすべてBOOTH行き。
- FAQの記述: 151行目「BOOTHから体験版をダウンロードしてご利用ください。」現状はBOOTH集約で一貫しており矛盾ではないが、試用版のReleases配置後は書き換え対象になる。
- 試用版の前提: ZIPが163.6MBで100MB制限を超えるため、公式HP直DL化はGitHub Releases方針の確定（版・ファイル名の決定）後。それまでは現状のBOOTH一貫導線を維持するのが安全。
- 重複セクション・id名の不一致: なし（id="trial" は体験版説明セクションで内容一致）。
- GA4タグ: なし。
- 修正方針案: ヒーローのボタン順（体験版アンカーを先、BOOTH購入を後）だけは配置前でも修正可能だが、試用版DL化と同時にやり直しになるため、Releases配置とセットで一括修正するのが効率的。
