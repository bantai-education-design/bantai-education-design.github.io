### 1-3. 文部科学省 学校コード

**出典URL**: https://www.mext.go.jp/b_menu/toukei/mext_01087.html
**検索サービス**: https://edu-data.jp/（文部科学省委託・チエル株式会社運営）

#### 学校コードの体系（13桁）

| 桁 | 内容 | 例（東京都・公立・小学校） |
| :--- | :--- | :--- |
| 1-2桁 | 学校種コード | B1（小）/ C1（中）/ D1（高）/ A1（幼）/ E1（特支）/ D2（中等） |
| 3-4桁 | 都道府県コード（JIS） | 13（東京都） |
| 5桁 | 設置区分 | 1=国立 / 2=公立 / 3=私立 |
| 6-12桁 | 学校番号（7桁連番） | 各学校固有 |
| 13桁 | チェックディジット | 入力誤り防止 |

#### 利用上の評価

| 項目 | 内容 |
| :--- | :--- |
| データ形式 | CSV / Excel（全国一括 or 都道府県別） |
| 東京都分のみ取得 | ✅ 可能（都道府県別ファイル） |
| 電話番号の収録 | ❌ 学校コードデータには含まれない場合あり |
| 住所の収録 | ✅ 収録あり |
| ライセンス | 政府標準利用規約（CC BY相当）出典明記で再利用可 |
| 更新頻度 | 年1回（5月1日基準） |
| 最新版 | 令和8年5月1日時点（暫定版が2026年時点で最新） |

> [!NOTE]
> 学校コードは「一意ID」としての利用価値が高い。ただし電話番号が含まれないため、
> 住所・電話番号は別途東京都公立/私立学校一覧データと突合が必要。
> edu-data.jpでは東京都・学校種を指定してExcel出力可能（5,000件以内）。

## 第6章 注意事項・原則

### 6-1. 出典とライセンス

| データ種別 | 出典 | ライセンス | 必須対応 |
| :--- | :--- | :--- | :--- |
| 公立学校一覧 | 東京都教育委員会 | CC BY 4.0 | 出典・基準日の明記 |
| 私立幼稚園一覧 | 東京都生活文化スポーツ局 | CC BY | 出典・基準日の明記 |
| 私立学校一覧 | 東京都生活文化スポーツ局 | CC BY | 出典・基準日の明記 |
| 学校コード | 文部科学省 | 政府標準利用規約（CC BY相当） | 出典明記、改変の旨記載 |

現在の検索ページに掲載済みの免責表示はそのまま維持し、追加データに応じて出典を追記する。

### 6-2. データ品質原則

- **推測補完禁止**: 住所・電話番号・ふりがなを推測で補完しない。出典データに存在しない情報は空欄またはフラグで管理する
- **個人情報の除外**: 校長名・教員名・児童生徒名・個人の連絡先は絶対に含めない
- **学校データと企業データの分離**: 大分類フィールドで「学校」と「教育関係機関」を明確に区別し、JSONも別ファイル管理を検討する

### 6-3. 既存機能の保全

- search.js の検索ロジック（キーワード・区市町村フィルター・学校種フィルター）は変更なし。フィルター追加のみで対応可能
- MUNICIPALITY_ORDER 配列（行政順62自治体）は維持する
- GA4イベント6種（school_search / school_municipality_filter / school_type_filter / school_address_copy / school_csv_download / envelope_app_click）は維持し、UIに応じて追加イベントを命名規則に沿って拡張
- CSVダウンロード・宛名コピー・敬称切り替え・上へ戻るボタンは既存仕様を維持

### 6-4. パフォーマンス設計

| フェーズ | 想定件数 | 対応 |
| :--- | :--- | :--- |
| 現状 | 1,869件 | 問題なし |
| Phase A後 | 約2,275件 | 問題なし（現行方式で対応可） |
| Phase B後 | 約3,650件 | 動作確認が必要。仮想スクロール・ページングの検討開始 |
| Phase C〜D後 | 4,000件超 | ページング or 検索結果の上限表示を推奨 |

## 第7章 データ取得・処理のロードマップ

### Phase A 作業手順（概要）

1. データ取得: 東京都教育委員会サイトから各学校種Excel/CSVをダウンロード（幼稚園・高校・中等・特支）
2. スキーマ確認: 各ファイルの列名・エンコーディング（Shift-JIS）を確認
3. process_full_dataset.py 拡張: 学校種別ファイルを統合し、新スキーマに変換
4. データ検証: 件数・住所・電話番号の完備率を確認
5. JSONマージ: 既存の1,869件のJSONに追加データを統合
6. UI追加: 学校種チェックボックス（幼稚園・高校・中等・特支）を追加
7. GA4イベント確認: school_type_filter イベントが新学校種に対応しているか確認
8. データベース名変更: ページタイトル・説明文・トップページ記載を更新
9. 出典表示更新: フッター・免責表示に追加データの出典・基準日を明記
10. 動作確認・PR作成 → mainへマージ

### Phase B 作業手順（概要）

1. データ取得: 東京都オープンデータカタログから私立幼稚園CSV・私立学校一覧CSVをダウンロード
2. CSVエンコーディング・項目名確認（Shift-JIS or UTF-8、列名の確認）
3. 設置区分フィールドの追加: 設置区分（公立/私立）を既存データにも遡及追加
4. JSONへの統合: 私立データを既存JSONにマージ（ファイル分割も検討）
5. UI追加: 設置区分フィルター（公立/私立/すべて）を追加
6. 出典表示更新: 生活文化スポーツ局データの出典・基準日を追記
7. 動作確認・PR作成 → mainへマージ
