# Ban.Tai 学校データベース 共通データ仕様

## 目的

東京都版の既存データを維持しながら、埼玉県以降の都道府県データを同じ検索UIへ追加できるようにする。

## 互換フィールド（検索画面で使用）

| フィールド | 必須 | 内容 |
|---|---:|---|
| `school_name` | 必須 | 学校・園の正式名称 |
| `school_name_kana` | 任意 | よみがな。原本にない場合は空文字 |
| `school_type` | 必須 | 幼稚園、小学校、中学校、義務教育学校、高等学校、中等教育学校、特別支援学校 |
| `establishment_type` | 必須 | 公立、私立、国立 |
| `municipality` | 必須 | 市区町村名 |
| `postal_code` | 必須 | 7桁ハイフン形式 |
| `address` | 必須 | 都道府県名から始まる住所 |
| `phone` | 任意 | 電話番号 |

## 拡張フィールド

| フィールド | 内容 |
|---|---|
| `id` | 都道府県内で一意のID |
| `prefecture` | 都道府県名 |
| `operator` | 設置者 |
| `course` | 全日制、定時制、通信制など |
| `branch_type` | 本校、分校、分教室など |
| `website` | 公式サイト |
| `source_name` | 一次資料名 |
| `source_url` | 一次資料ページURL |
| `source_date` | 原本の基準日 |
| `verified_date` | Ban.Taiでの確認日 |

## 埼玉県初版のルール

1. 原本は埼玉県・埼玉県教育委員会の公開資料だけを使用する。
2. 学校名、住所、電話番号を推測で補わない。
3. 公立高校は課程別Excelを読み込むが、同一校・同一住所・同一電話番号は原則1校へ統合し、課程を `course` に保持する。
4. 分校・分教室は所在地または電話番号が本校と異なる場合、独立レコードとして保持する。
5. 休校中の学校は原本の掲載基準に従い、公開対象から除外する。
6. 認定こども園、専修学校、各種学校、大学は初版対象外とする。

## 検証基準

- 必須項目の空欄ゼロ
- `id` の重複ゼロ
- 郵便番号は `000-0000`
- 住所は `埼玉県` で始まる
- 完全一致重複ゼロ
- 校種別・設置区分別の件数を公式資料と照合
- 各区分から抽出した100件以上を原本と手動照合
